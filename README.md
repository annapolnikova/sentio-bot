# Sentio Test Group Bot

Telegram-бот для тестовой группы Sentio:
- регистрирует участниц (ім'я, email, телефон, вік, який рол-он тестує);
- щодня о заданий час (за замовчуванням 20:00 Kyiv) надсилає коротку анкету прямо в чат — питання ті самі, що й у Google Form;
- нагадує лише тим, хто ще не відповів сьогодні;
- дозволяє тобі (адміну) в будь-який момент вивантажити всі відповіді в Excel командою `/export`.

Питання анкети: стан, відчуття (мультивибір), час ефекту, оцінка аромату, частота використання, вільний опис досвіду, чи продовжить користуватись, чи можна цитувати відгук, Instagram.

---

## 1. Створити бота в Telegram

1. Напиши в Telegram боту **@BotFather** → `/newbot`.
2. Дай ім'я та юзернейм (напр. `SentioFeedbackBot`).
3. BotFather дасть токен виду `123456789:AA...` — це `BOT_TOKEN`.
4. Дізнайся свій Telegram ID: напиши **@userinfobot** — він покаже твій ID (це `ADMIN_IDS`).

## 2. Розгорнути на Railway (безкоштовно)

Railway дає безкоштовний ліміт, якого вистачить для тестової групи на кілька десятків людей.

1. Зареєструйся на **railway.app** (можна через GitHub).
2. Створи новий репозиторій на GitHub і залий туди всі файли з цієї папки (`bot.py`, `requirements.txt`, `Procfile`).
   - Найпростіше: зайди на github.com → New repository → Upload files → перетягни файли.
3. В Railway: **New Project → Deploy from GitHub repo** → обери свій репозиторій.
4. Railway сам розпізнає Python-проєкт і встановить залежності з `requirements.txt`.
5. В розділі **Variables** додай змінні середовища (значення зі свого `.env.example`):
   - `BOT_TOKEN` — токен від BotFather
   - `ADMIN_IDS` — твій Telegram ID
   - `TZ_NAME` — `Europe/Kyiv`
   - `REMINDER_HOUR` — `20`
   - `REMINDER_MINUTE` — `0`
   - `DB_PATH` — `sentio.db`
6. В розділі **Settings → Deploy** переконайся, що команда запуску — `python bot.py` (Railway підхопить це з `Procfile` автоматично).
7. Натисни **Deploy**. За хвилину бот стане онлайн — перевір в Telegram командою `/start`.

**Важливо про базу даних:** на безкоштовному тарифі Railway файлова система може очищатись при кожному новому деплої коду. Щоб дані (реєстрації + відповіді) зберігались назавжди:
- В Railway додай **Volume** (Settings → Volumes → New Volume, mount path напр. `/data`) і встанови `DB_PATH=/data/sentio.db`.
- Або просто регулярно роби `/export` — так у тебе завжди буде свіжа копія в Excel.

## 3. Команди бота

**Для учасниць:**
- `/start` — реєстрація
- `/survey` — заповнити анкету вручну (окрім щоденного нагадування)
- `/skip` — пропустити необов'язкове текстове поле
- `/cancel` — скасувати поточну дію

**Для адміна (тебе):**
- `/stats` — скільки зареєстровано, скільки відповідей всього і за сьогодні
- `/export` — отримати Excel-файл з двома листами: Users (учасниці) і Responses (усі відповіді з датами)
- `/whoami` — дізнатись свій Telegram ID

## 4. Локальний запуск (якщо захочеш протестувати на своєму компʼютері)

```bash
pip install -r requirements.txt
export BOT_TOKEN="токен_від_BotFather"
export ADMIN_IDS="твій_telegram_id"
python bot.py
```

## 5. Якщо захочеш змінити питання анкети

Усі тексти питань і варіантів відповідей знаходяться на початку файлу `bot.py` у змінних
`STATE_OPTIONS`, `FEEL_OPTIONS`, `TIME_OPTIONS`, `AROMA_OPTIONS`, `FREQ_OPTIONS`,
`CONTINUE_OPTIONS`, `CONSENT_OPTIONS` — можна редагувати прямо там.
